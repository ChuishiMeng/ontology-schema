## 2. 核心算法设计

### 2.1 Ontology-Schema 统一表示

#### 2.1.1 统一知识图谱构建

统一知识图谱 (UKG) 将 Ontology 实体、Schema 表/列、对齐关系整合为异构图：

- **节点类型**: Ontology实体、Schema表、Schema列
- **边类型**: Ontology关系、Schema外键、对齐关系

#### 2.1.2 异构图神经网络编码

使用 HeteroSchemaGNN 对统一知识图谱进行编码：

1. **节点特征初始化**: Ontology实体用名称+类型+描述，Schema表用表名+列名+类型+外键
2. **异构图卷积**: 3层卷积，同类型节点信息传递 → 跨类型信息传递 → 全局上下文聚合
3. **联合表示输出**: 输出对齐的向量空间，支持跨模态检索

#### 2.1.3 对齐训练策略

使用对比学习损失进行对齐训练：

$$
\mathcal{L}_{align} = -\log \frac{\exp(sim(h_i^{ont}, h_j^{schema}) / \tau)}{\sum_{k} \exp(sim(h_i^{ont}, h_k^{schema}) / \tau)}
$$

---

### 2.2 联合召回机制

#### 2.2.1 三阶段召回流程

**Stage 1: 语义召回** (Top-50)
- Query编码 → 向量检索 → 语义相似度计算
- 信号: $S_{sem}(q, e) = \cos(\vec{q}, \vec{e}_{schema})$

**Stage 2: 结构扩展**
- 外键传播: 通过FK关系扩展相关表
- 路径推理: BFS找已知表到目标实体的最短路径

**Stage 3: 规则增强**
- 业务规则匹配: 匹配查询中的业务概念
- 隐含关系注入: 补充业务知识

#### 2.2.2 多信号分数融合

最终分数：

$$
S_{final}(q, e) = \alpha \cdot S_{sem}(q, e) + \beta \cdot S_{struct}(q, e) + \gamma \cdot S_{rule}(q, e)
$$

自适应权重根据查询类型调整：
- simple_lookup: sem=0.7, struct=0.2, rule=0.1
- join_query: sem=0.4, struct=0.5, rule=0.1
- aggregation: sem=0.3, struct=0.3, rule=0.4
- business_analysis: sem=0.3, struct=0.2, rule=0.5

---

### 2.3 评分与排序策略

#### 2.3.1 重排序模型

使用 Cross-Encoder 重排序，结合多维特征：
- 语义特征: cosine_sim, entity_overlap
- 结构特征: fk_distance, path_length
- 规则特征: rule_match, concept_relevance
- 统计特征: table_size, column_null_rate

### 3.2 核心模块

#### Unified Knowledge Base
- **Ontology Store**: 实体、关系、业务规则、自然语言描述
- **Schema Store**: 表、列、外键、约束、统计信息
- **Alignment Index**: 统一嵌入索引 (FAISS) + 图结构索引 (NetworkX)

#### Joint Retrieval Module
- **Semantic Recaller**: 语义向量检索
- **Structural Recaller**: FK传播、路径推理
- **Rule Recaller**: 业务规则匹配
- **Adaptive Fusion**: 自适应权重融合
- **Reranker**: Cross-Encoder重排序

### 3.3 数据流

```
Query: "找出去年购买金额最高的前10位客户"

1. 意图理解: aggregation + ranking, entities=[客户, 购买金额], time=去年

2. 联合召回:
   - 语义召回 → Top-50 tables/columns
   - 结构扩展 → customers → orders (via FK)
   - 规则增强 → "去年"=fiscal_year_rule, "购买金额"=total_amount_rule
   - 融合排序 → [customers, orders, order_items]

3. SQL生成:
   Prompt包含:
   - 可用表: customers, orders
   - 业务规则: 去年=fiscal year, 购买金额=SUM(total_amount)
   - LLM生成SQL

4. 验证执行:
   - 语法验证 ✓
   - Schema约束验证 ✓
   - 执行返回结果
```

prompt = f"""<instruction>

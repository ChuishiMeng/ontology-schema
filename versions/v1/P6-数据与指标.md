## 2. 评估指标

### 2.1 召回层指标

#### 2.1.1 Recall@K

定义：正确的召回结果在前K个中的比例

$$
\text{Recall@K} = \frac{1}{|Q|}\sum_{q \in Q} \frac{|R_q \cap G_q|}{|G_q|}
$$

其中:
- $Q$: 测试查询集合
- $R_q$: 系统召回的前K个结果
- $G_q$: 查询$q$的真实相关结果

**目标值**: Recall@5 > 0.90

#### 2.1.2 Precision@K

定义：前K个召回结果中正确的比例

$$
\text{Precision@K} = \frac{1}{|Q|}\sum_{q \in Q} \frac{|R_q \cap G_q|}{K}
$$

**目标值**: Precision@5 > 0.85

#### 2.1.3 MRR (Mean Reciprocal Rank)

定义：第一个正确结果的排名倒数的均值

$$
\text{MRR} = \frac{1}{|Q|}\sum_{q \in Q} \frac{1}{\text{rank}_q}
$$

**目标值**: MRR > 0.88

#### 2.1.4 NDCG (Normalized Discounted Cumulative Gain)

定义：考虑位置衰减的排序质量指标

$$
\text{NDCG@K} = \frac{1}{|Q|}\sum_{q \in Q} \frac{DCG_q@K}{IDCG_q}
$$

**目标值**: NDCG@5 > 0.85

### 2.2 生成层指标

#### 2.2.1 SQL Valid Rate

定义：生成的SQL语法正确且可执行的比例

$$
\text{Valid Rate} = \frac{\#\text{valid SQL}}{\#\text{total SQL}}
$$

**目标值**: > 0.95

#### 2.2.2 Execution Accuracy (Exec Acc)

定义：SQL执行结果与Gold SQL完全一致的比例

$$
\text{Exec Acc} = \frac{\#\text{exactly match}}{\#\text{total}}
$$

这是Spider/BIRD的主要评估指标。

**目标值**: > 0.85

#### 2.2.3 Exact Match Accuracy

定义：生成的SQL与Gold SQL字符串完全匹配的比例

**特点**: 严格指标，但允许等价SQL

### 2.3 端到端指标

#### 2.3.1 Task Completion Rate

定义：完整任务流程成功的比例

```
Success Criteria:
1. 召回相关表/列 (Recall@3 >= 0.8)
2. 生成有效SQL
3. SQL执行无错误
4. 返回非空结果
```

**目标值**: > 0.80

#### 2.3.2 Latency

| 阶段 | 目标延迟 |
|------|----------|
| 召回阶段 | < 500ms |
| 生成阶段 | < 1500ms |
| 端到端 | < 2000ms |

### 2.4 对齐质量指标

#### 2.4.1 向量空间对齐度

定义：Ontology和Schema向量空间的相似度

$$
\text{Align Score} = \text{cosine}(h^{ont}, h^{schema})
$$

**目标值**: > 0.85

## 3. 评测流程

### 3.1 召回评估流程

```
1. 加载测试查询
2. 对每个查询:
   a. 提取查询意图和实体
   b. 执行召回流程
   c. 获取Top-K结果
   d. 与Ground Truth比较
3. 计算指标均值
```

### 3.2 生成评估流程

```
1. 加载测试查询 + Schema
2. 对每个查询:
   a. 执行召回获取Context
   b. 调用LLM生成SQL
   c. 验证SQL有效性
   d. 执行SQL与Gold对比
3. 计算Exec Acc等指标
```

## 4. 数据格式

### 4.1 测试数据格式 (JSON)

```json
{
  "query_id": "spider_1001",
  "question": "找出平均分数最高的前5名学生",
  "db_id": "school",
  "gold_sql": "SELECT student_name, AVG(score) FROM scores GROUP BY student_name ORDER BY AVG(score) DESC LIMIT 5",
  "difficulty": "medium",
  "query_type": "aggregation",
  "ontology": {
    "entities": ["学生", "分数"],
    "relations": ["选修", "拥有分数"]
  },
  "business_rules": [
    "平均分数=AVG(score)",
    "前5名=LIMIT 5"
  ]
}
```

### 4.2 对齐数据格式

```json
{
  "alignment_id": "align_001",
  "ontology_entity": {
    "id": "ont_student",
    "name": "学生",
    "type": "entity",
    "description": "在学校注册并学习的个体"
  },
  "schema_element": {
    "id": "students",
    "name": "students",
    "type": "table",
    "columns": ["student_id", "student_name", "enrollment_date"]
  },
  "alignment_type": "entity-table",
  "confidence": 1.0
}
```

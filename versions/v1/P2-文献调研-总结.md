## 🔬 方法论总结

### Schema Linking 方法分类

| 方法类型 | 代表论文 | 优点 | 缺点 |
|----------|---------|------|------|
| **语义相似度** | DIN-SQL, PET-SQL | 简单高效 | 召回有限 |
| **双向链接** | RSL-SQL | 高召回 (94%) | 多次 LLM 调用 |
| **图路径算法** | SchemaGraphSQL | 零样本、低成本 | 依赖外键 |
| **多智能体** | LinkAlign, MAG-SQL | 可解释、鲁棒 | 成本高 |

### 关键技术要素

| 要素 | 来源论文 | 应用场景 |
|------|---------|---------|
| **Query Rewriting** | LinkAlign | 语义对齐 |
| **多智能体辩论** | LinkAlign, MAG-SQL | 验证和消歧 |
| **软选择** | MAG-SQL | 降低遗漏风险 |
| **图路径** | SchemaGraphSQL | 连接推理 |

## 🎯 研究空白

### 现有方法局限

| 局限 | 具体表现 |
|------|---------|
| **无 Ontology 层** | 无法处理业务术语与 Schema 的语义鸿沟 |
| **依赖外键** | SchemaGraphSQL 等需要完整外键约束 |
| **未融合业务知识** | 无法利用 Wiki、指标字典等非结构化知识 |

### 我们的机会

| 差异化点 | 价值 |
|----------|------|
| **三层图谱架构** | L0 元模型 + L1 业务层 + L2 物理层 |
| **Ontology 增强** | 业务术语桥接 |
| **元路径约束推理** | 保证结构完整性 |

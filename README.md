# Ontology-Schema 联合召回

> **研究目标**：构建业务知识与数据库 Schema 的联合召回框架，解决 Text-to-SQL 中的语义鸿沟
> **核心定位**：将 OBDA（Ontology-Based Data Access）思想迁移到 LLM 时代
> **目标会议**：VLDB/SIGMOD > KDD > EMNLP
> **启动日期**：2026-03-12
> **当前阶段**：P7 实验验证（可启动）

---

## 研究概要

业务人员用业务术语查询数据时，存在严重的语义鸿沟——他们不知道数据库表结构，也不了解 SQL 语法。现有 Text-to-SQL 方案停留在表面级 Schema Linking，无法弥合业务知识与数据库 Schema 之间的深层语义差异。

本项目提出**异构图谱驱动的最小完备集提取**方案：
1. 用 LLM 半自动构建领域 Ontology（业务术语 → 数据库 Schema 的语义映射）
2. 通过两层图谱（业务层 + Schema 层）实现联合召回
3. 基于元路径推理提取最小完备集，增强 LLM 的 Schema 理解

## 核心差异化

| 维度 | OBDA 传统方案 | BIRD Knowledge Evidence | 本项目 |
|------|--------------|------------------------|--------|
| 知识表示 | 形式化 Ontology | 非结构化文本 | 结构化图谱 |
| 构建方式 | 人工/专家 | 人工标注 | LLM 半自动 |
| 推理能力 | 强（DL 推理） | 弱（文本匹配） | 中（元路径推理） |
| LLM 兼容 | 不兼容 | 原生兼容 | LLM 增强 |

## 🚨 关键发现：ATHENA (VLDB 2016)

**Saha et al. "ATHENA: An Ontology-Driven System for Natural Language Querying over Relational Data Stores"**

这篇 **10 年前的 VLDB 论文** 几乎做了同样的事情：

| 维度 | ATHENA (2016) | 本研究 (2026) |
|------|---------------|--------------|
| **核心思路** | Ontology 驱动 NL→SQL | Ontology 驱动 NL→SQL |
| **两阶段** | NL→OQL→SQL | NL→联合召回→SQL |
| **Ontology** | OWL2，手工构建 | LLM 半自动构建 |
| **映射** | Ontology-to-Database Mapping | L1→L2 映射 |
| **Ontology 层** | ✅ 有（概念+属性+关系+继承） | ✅ 有（L1 Ontology） |
| **Schema 层** | ✅ 有（关系数据库） | ✅ 有（L2 Schema） |
| **性能** | GEO 100% precision, FIN 99% precision | 未实验 |
| **不含 LLM** | 传统 NLP（规则+同义词匹配） | 基于 LLM |

## 🎯 重新定位策略

### ❌ 不能再说的话
- ~~"现有方法均未引入 Ontology 层"~~ — ATHENA 10 年前就做了
- ~~"首次将 Ontology 与 Schema 融合"~~ — OBDA 研究了 20 年
- ~~"首次设计 Ontology-Schema 联合召回"~~ — ATHENA 的两阶段本质上就是

### ✅ 应该说的差异化

**核心定位：LLM 时代的 Ontology-Augmented Schema Linking**

本研究与 ATHENA 等 OBDA 系统的**关键区别**在于：

| 维度 | ATHENA/OBDA 范式 | 本研究 |
|------|-----------------|--------|
| **技术时代** | 传统 NLP（规则+同义词） | LLM-based |
| **核心问题** | NL→Ontology 理解 | Schema Linking + 知识联合召回 |
| **Ontology 构建** | 完全手工（领域专家） | LLM 半自动 + 人工校验 |
| **召回策略** | 无（全量 Schema） | 最小完备集提取 |
| **推理机制** | 无（基于同义词映射） | 元路径约束推理 |
| **评估场景** | 小规模自定义数据集 | BIRD 等标准 Benchmark |
| **可扩展性** | 低（强依赖 Ontology 质量） | 高（Ontology 不完整时有 fallback） |

### 重新定义的创新点

| 优先级 | 创新点 | 描述 |
|--------|--------|------|
| ⭐⭐⭐⭐⭐ | **LLM-driven Ontology 半自动构建** | 从 Schema + 业务文档自动生成 Ontology，降低构建成本 |
| ⭐⭐⭐⭐⭐ | **Ontology-Schema 联合召回 + 最小完备集** | OBDA 全量查询 vs 我们的精准召回 |
| ⭐⭐⭐⭐ | **元路径约束推理** | OBDA 无此机制，我们利用图结构做语义约束 |
| ⭐⭐⭐ | **在标准 Benchmark (BIRD) 上评估** | ATHENA/Sequeda 用自定义数据集，我们在社区标准上对比 |

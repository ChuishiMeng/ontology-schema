## 📋 基本信息

**研究主题**: 小模型 Text-to-SQL 分解微调方法
**核心问题**: 开源小模型 vs 闭源大模型的性能差距
**研究领域**: CS/数据库/NLP
**作者机构**: 阿尔伯塔大学

---

## 🎯 研究背景与问题

### 核心矛盾

| 对比 | 模型 | EX | EM |
|------|------|-----|-----|
| **Fine-tuning (开源)** | Llama2 7B | 66.7% | 63.9% |
| **Prompting (闭源)** | GPT-4 + DAIL-SQL | 84.4% | 74.4% |
| **差距** | - | **~18%** | **~10%** |

### 问题分析

传统微调方法同时训练两个目标：
1. **Schema Linking**：识别相关表和列
2. **SQL Generation**：生成正确 SQL

**问题**：两个任务都需要独立的推理过程，小模型难以同时学习。

## 🔬 核心框架

### DTS-SQL 两阶段分解

```
┌─────────────────────────────────────────┐
│        DTS-SQL 框架                      │
├─────────────────────────────────────────┤
│                                          │
│  阶段1: Schema-Linking Fine-tuning       │
│  ├── 明确训练 Schema Linking            │
│  ├── 输入: 问题 + 全部 Schema            │
│  └── 输出: 相关表和列                    │
│                                          │
│  阶段2: SQL Generation Fine-tuning       │
│  ├── 输入: 问题 + 选中的 Schema          │
│  └── 输出: SQL 查询                      │
│                                          │
└─────────────────────────────────────────┘
```

### 关键创新

| 创新 | 说明 |
|------|------|
| **分解微调** | 两阶段分别微调 |
| **Schema Linking 显式训练** | 作为独立任务 |
| **小模型可用** | 7B 参数也能达到好效果 |

### 公式化

**传统方法**：
$$\min_{\sigma, M^*} \frac{1}{|T|} \sum_{i=1}^{|T|} L(M^*(\sigma_f(q_i, D_i, s_i)))$$

**问题**：同时学习 Schema Linking + SQL 生成

**DTS-SQL**：分解为两个阶段，分别优化

## 📊 实验结果

### Spider Test Set

| 方法 | 模型 | EX | EM |
|------|------|-----|-----|
| DAIL-SQL + GPT-4 | 闭源 | 86.6% | - |
| DIN-SQL + GPT-4 | 闭源 | 85.3% | 60% |
| **DTS-SQL + DeepSeek 7B** | **开源** | **84.4%** | **73.7%** ⭐ |
| RESDSQL-3B + NatSQL | 开源 | 79.9% | 72% |
| **DTS-SQL + Mistral 7B** | **开源** | **77.1%** | **69.3%** |

### 关键对比

| 模型 | Fine-tuning | DTS-SQL | 提升 |
|------|-------------|---------|------|
| **Mistral 7B** | 71.9% EX | **78.6% EX** | **+6.7%** |
| **DeepSeek 7B** | 82.1% EX | **85.5% EX** | **+3.4%** |

### Upper Bound 分析

| 模型 | Upper Bound (Gold Schema) |
|------|---------------------------|
| Mistral 7B | 86.6% EX |
| DeepSeek 7B | 90.3% EX |

**说明**：如果 Schema Linking 完美，性能可进一步提升。

## 💡 研究贡献与局限

### 核心贡献

| 贡献 | 说明 |
|------|------|
| **分解微调** | 首次明确提出两阶段分解微调 |
| **小模型突破** | 7B 模型接近 GPT-4 水平 |
| **显式 Schema Linking** | 作为独立任务训练 |

### 局限性

1. **两阶段推理**：增加推理时间
2. **依赖训练数据**：需要标注数据
3. **未涉及 Ontology**：无法处理业务术语

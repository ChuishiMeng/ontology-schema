## 📋 基本信息

**研究主题**: 多智能体 TableQA 框架
**核心问题**: 结构化数据 + 知识图谱的协同问答

**作者机构**: 军事科学院、人民大学

## 🎯 研究背景与问题

### 现有方法局限

| 挑战 | 说明 |
|------|------|
| **上下文长度限制** | 限制数据处理能力 |
| **幻觉问题** | 影响答案可靠性 |
| **单智能体架构** | 难以处理复杂推理 |

### 研究动机

复杂问答需要：
- 结构化推理（SQL）
- 关系推理（KG 多跳）
- 两者协同

```
┌─────────────────────────────────────────┐

│                                          │

│                                          │
└─────────────────────────────────────────┘
```

### 关键创新

| 创新 | 说明 |
|------|------|
| **Data-to-KG 自动转换** | 关系数据 → 知识图谱 |
| **自然语言协商** | 智能体间灵活协作 |
| **Context Engineering** | 减少幻觉，提升准确率 |

## 📊 实验结果

### 性能提升

| 数据集 | 提升幅度 |
|--------|---------|
| **TabFact** | +20.2% |
| **WikiTQ** | +23.9% |

### 团队协作效果

| 对比 | TabFact | WikiTQ | FeTaQA |
|------|---------|--------|--------|
| 双团队 vs 单团队 | +5.5% | +14.4% | +17.1% |

### 关键发现

1. **Database + KG 协作有效**：互补优势
2. **自然语言协商优于固定流程**：灵活适应
3. **Context Engineering 减少幻觉**

---

## 💡 研究贡献与局限

### 核心贡献

1. **Data-to-KG 转换形式化**：映射函数定义
2. **多智能体协作**：Database + KG 双团队
3. **自然语言协商机制**：灵活智能体交互
4. **显著性能提升**：TabFact +20.2%，WikiTQ +23.9%

### 局限性

1. **TableQA 专注**：未涉及完整 Text-to-SQL
2. **依赖 LLM**：协商依赖 LLM 能力
3. **映射函数简化**：未涉及 Ontology 层

## 📋 基本信息

**研究主题**: LLM 在结构化数据上的推理框架
**核心问题**: 如何让 LLM 在知识图谱、表格、数据库等结构化数据上进行推理
**研究领域**: CS/知识图谱/NLP
**作者机构**: 人民大学高瓴人工智能学院

---

## 🎯 研究背景与问题

### 核心问题

LLM 的局限性：
| 问题 | 说明 |
|------|------|
| **事实错误** | 生成与事实冲突的信息 |
| **缺乏领域知识** | 无法掌握领域特定或实时知识 |
| **结构化数据处理困难** | 无法直接处理 KG、表格、数据库 |

### 研究动机

结构化数据（KG、Table、DB）是知识的重要载体，但 LLM 难以直接处理其特殊格式。

---

## 🔬 核心框架

### StructGPT 框架

```
┌─────────────────────────────────────────┐
│        StructGPT 框架                    │
├─────────────────────────────────────────┤
│                                          │
│  Iterative Reading-then-Reasoning (IRR)  │
│                                          │
│  1. Reading Phase                        │
│     └── 专用接口收集相关证据             │
│                                          │
│  2. Reasoning Phase                      │
│     └── LLM 基于收集信息推理             │
│                                          │
│  3. Invoking-Linearization-Generation    │
│     └── 迭代执行直到得到答案             │
│                                          │
└─────────────────────────────────────────┘
```

### 关键组件

| 组件 | 作用 |
|------|------|
| **Specialized Interfaces** | 为 KG/Table/DB 设计专门接口 |
| **Invoking** | 调用接口获取结构化数据 |
| **Linearization** | 将结构化数据线性化为文本 |
| **Generation** | LLM 基于文本生成答案 |

### 支持的结构化数据类型

| 类型 | 接口示例 |
|------|---------|
| **Knowledge Graph** | 实体查询、关系遍历 |
| **Table** | 行选择、列投影 |
| **Database** | SQL 查询 |

## 📊 实验结果

### 知识图谱问答

| 方法 | 数据集 | 性能 |
|------|--------|------|
| LLM Zero-shot | - | 基线 |
| **StructGPT** | **提升显著** | Few-shot/Zero-shot 都有效 |

### 表格问答

| 方法 | WikiTableQuestions | TabFact |
|------|-------------------|---------|
| LLM Baseline | - | - |
| **StructGPT** | **提升** | **提升** |

### 关键发现

1. **迭代推理有效**：多轮 Reading-Reasoning 提升准确率
2. **接口设计关键**：好的接口能收集更相关证据
3. **Few-shot 可用**：少量示例即可提升

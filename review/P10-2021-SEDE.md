## 🎯 研究背景与问题

### 现有数据集局限

| 问题 | 说明 |
|------|------|
| **人造数据** | 非 natural occurring，问题和 SQL 都是专门编写的 |
| **缺乏多样性** | 问题模式固定，风格单一 |
| **不反映真实** | 用户实际提问更复杂、更多样 |

### 真实世界挑战

| 挑战 | 学术数据集 | 真实世界 (SEDE) |
|------|-----------|----------------|
| SQL 复杂度 | 相对简单 | 更复杂（嵌套、窗口函数等）|
| 问题风格 | 标准化 | 多样化、口语化 |
| Schema 理解 | 问题明确提及列 | 问题可能不提及列名 |

## 🔬 核心贡献

### SEDE 数据集

| 属性 | 数值 |
|------|------|
| **规模** | 12,023 对 (问题, SQL) |
| **来源** | Stack Exchange 真实使用 |
| **特点** | 自然产生的问题 |

### 数据集特点

```
真实世界示例:

问题: "Questions which attract bad answers"
描述: "posts which have attracted significantly more controversial or bad answers than good ones"

SQL: 
SELECT p.Id as [Post Link], p.Score from (
  SELECT p.ParentId, count(*) as ContACnt from (
    SELECT PostId,
      up = sum(case when VoteTypeId = 2 then 1 else 0 end),
      down = sum(case when VoteTypeId = 3 then 1 else 0 end)
    FROM Votes v join Posts p on p.Id = v.PostId
    WHERE VoteTypeId in (2,3) and PostTypeId = 2
    group by PostId
  ) ...
)
```

**特点**：
- 问题不直接提及表名/列名
- SQL 复杂，包含嵌套子查询
- 需要领域知识理解

## 📊 实验结果

### 模型性能对比

| 方法 | Spider | SEDE |
|------|--------|------|
| T5-Base | ~70% | **显著下降** |
| RAT-SQL | ~65% | **显著下降** |

**结论**：现有方法在真实世界数据上性能大幅下降

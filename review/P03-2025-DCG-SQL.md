## 📋 基本信息

**研究主题**: 基于 Schema Link Graph 的示例检索
**核心问题**: 如何有效表示和检索 Text-to-SQL 示例
**研究领域**: CS/数据库/NLP
**作者机构**: 成均馆大学

---

## 🎯 研究背景与问题

### 核心问题

现有 In-Context Learning 方法局限：
- 仅基于问题文本检索示例
- 忽略 Schema 信息
- 随机选择示例效果相当

### 关键发现

| 方法 | 检索 | GPT-4 | Llama 3.1-8B |
|------|------|-------|--------------|
| ACT-SQL | ✓ | 83.9% | 75.0% |
| ACT-SQL | ✗ | 83.2% | 75.6% |
| DAIL-SQL | ✓ | 82.1% | 69.9% |
| DAIL-SQL | ✗ | 80.8% | 69.3% |

**结论**：现有检索方法无效，与随机选择相当

### 研究动机

Schema 决定 SQL 的结构和语义，必须在示例表示中考虑。

---

## 🔬 核心框架

### Deep Contextual Schema Link Graph (DCG)

**核心思想**：构建问题与 Schema 的联合图表示

```
DCG 结构：
- 节点: 问题 Token + Schema 项（表/列）
- 边: 语义关系（Question-Schema Link）
```

### 关键特性

| 特性 | 说明 |
|------|------|
| **联合表示** | 问题 + Schema 统一编码 |
| **图结构** | 捕获语义关系 |
| **Schema Link** | 问题 Token 与 Schema 项的连接 |

### 流程

1. 构建 DCG → 问题与 Schema 的图
2. 图编码 → GNN 编码
3. 检索示例 → 基于图相似度
4. 生成 SQL → In-Context Learning

## 📊 实验结果

### Spider Benchmark

| 方法 | GPT-4 | Llama 3.1-8B |
|------|-------|--------------|
| Random | 83.2% | 75.6% |
| ACT-SQL | 83.9% | 75.0% |
| **DCG-SQL** | **提升** | **提升** |

### 关键发现

1. 小模型更依赖示例质量
2. Schema 感知检索有效
3. 图表示优于纯文本

---

## 💡 研究贡献与局限

### 核心贡献

1. **联合表示**：问题 + Schema 图表示
2. **有效检索**：真正有价值的示例检索
3. **小模型友好**：对小模型提升更明显

### 局限性

1. **计算开销**：GNN 编码增加成本
2. **Schema Link 依赖**：需要准确的问题-Schema 对齐
3. **未涉及 Ontology**：无业务知识层

## 🌟 与我们研究的关系

### 相关性

| 维度 | DCG-SQL | 我们的方案 |
|------|---------|-----------|
| 表示方式 | 图表示 | 三层图谱 |
| Schema Link | 内部机制 | 核心任务 |
| 检索方式 | 图相似度 | 元路径推理 |

### 可借鉴

- ✅ 图结构表示问题-Schema 关系
- ✅ 联合编码而非分离编码

### 差异化

| 我们的独特价值 |
|---------------|
| 三层结构分离业务知识（L1）和物理层（L2）|
| Ontology 提供业务语义桥接 |
| 不依赖示例检索，直接推理 |

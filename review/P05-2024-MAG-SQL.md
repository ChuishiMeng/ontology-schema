| 问题 | 说明 |

## 🔬 核心框架

### MAG-SQL 四个 Agent

```
┌─────────────────────────────────────────┐
│           MAG-SQL Workflow               │
├─────────────────────────────────────────┤
│                                          │
│  1. Soft Schema Linker                   │
│     └── 软选择 + Entity-based 方法       │
│                                          │
│  2. Targets-Conditions Decomposer        │
│     └── 细粒度问题分解                   │
│                                          │
│  3. Sub-SQL Generator                    │
│     └── 迭代生成子 SQL                   │
│                                          │
│  4. Sub-SQL Refiner                      │
│     └── 自纠正错误                       │
│                                          │
└─────────────────────────────────────────┘
```

### 关键创新

| 组件 | 创新 |
|------|------|
| **Soft Schema Linking** | 软选择而非硬选择，Entity-based 增强 |
| **Targets-Conditions Decomposition** | 目标-条件分解，细粒度 |
| **Iterative Generating** | Sub-SQL 逐步生成 |
| **External Oversight** | 每步有 Refiner 验证 |

## 📊 实验结果

### BIRD Benchmark

| 方法 | 执行准确率 |
|------|-----------|
| Vanilla GPT-4 | 46.35% |
| MAC-SQL | 57.56% |
| **MAG-SQL + GPT-4** | **61.08%** ⭐ |

### 关键发现

1. **多智能体协作有效**：比单一模型提升显著
2. **软 Schema Linking 更好**：避免硬选择的风险
3. **迭代生成更可控**：每步有验证

---

## 💡 研究贡献与局限

### 核心贡献

1. **多智能体框架**：四个协作 Agent
2. **Soft Schema Linking**：软选择降低风险
3. **迭代生成**：逐步验证，外部监督
4. **BIRD 61.08%**：突破 60% 门槛

### 局限性

1. **多 Agent 成本高**：多次 LLM 调用
2. **未涉及 Ontology**：无业务知识层
3. **问题分解依赖 LLM**：可能分解错误

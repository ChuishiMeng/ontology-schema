## 🔬 新增论文深度分析

### P23: ATHENA (VLDB 2016) ⭐⭐⭐⭐⭐ 最关键先驱

**核心贡献**：
- 两阶段架构：NL → OQL (Ontology Query Language) → SQL
- Ontology 驱动的语义消歧
- Translation Index (TI)：数据值 + 同义词 + 语义变体索引
- Ontology-to-Database Mapping：手工定义

**关键数据**：
- FIN 领域：75 concepts, 289 properties, 95 relations
- Precision: GEO 100%, MAS 100%, FIN 99%
- Recall: 87-89%

**与本研究区别**：
| 维度 | ATHENA | 本研究 |
|------|--------|--------|
| Ontology 构建 | 手工 OWL2 | LLM 半自动 |
| NL 理解 | 传统 NLP（规则+同义词） | LLM |
| 召回策略 | 全量 Schema | 最小完备集 |
| 推理机制 | ontology metrics 排序 | 元路径约束 |
| 评估 | 自定义小数据集 | BIRD 标准 |

---

### P16: Sequeda et al. (GRADES 2023) ⭐⭐⭐⭐⭐ KG 影响评估

**核心贡献**：
- Benchmark：保险领域 13 表 + 43 问题 + OWL Ontology + R2RML
- 对比：GPT-4 直接查 SQL (16.7%) vs KG (54.2%) — 提升 3x

**关键发现**：
| Schema 复杂度 | SQL 准确率 | KG 准确率 |
|---------------|-----------|----------|
| ≤4 表 (Low) | 25-37% | 66-71% |
| >5 表 (High) | **0%** | 35-38% |

**启示**：
- ✅ KG 显著提升准确率（可作为本研究 Motivation）
- ✅ 高 Schema 复杂度时全量 Schema 完全失效 → 支持最小完备集必要性
- ⚠️ 但 Sequeda 只做评估，无方法创新

---

### P19: BIRD (NeurIPS 2023) ⭐⭐⭐⭐⭐ 最大真实基准

**核心贡献**：
- 12,751 text-to-SQL pairs，95 databases，33.4 GB
- 37 专业领域
- **Knowledge Evidence 机制**：外部知识 K 辅助理解数据库值

**公式化**：
$$Y = f(Q, D, K | \theta)$$

其中 K 是 external knowledge evidence（如"POPLATEK TYDNE stands for weekly issuance"）

**启示**：
- ✅ BIRD 的 knowledge evidence 本质上是简化版 Ontology
- ⚠️ 但它是非结构化文本片段，不支持结构化推理
- ⚠️ 本研究需解释与 BIRD knowledge evidence 的区别

## 🎯 研究空白分析（基于补充文献）

### G1: OBDA 未迁移到 LLM 时代（修订）

**证据**：
- ATHENA (2016) 已做 Ontology-driven NL→SQL
- Sequeda (2023) 已评估 KG 对 Text-to-SQL 的增益
- 但两者都用传统方法（手工 Ontology、传统 NLP）
- LLM-based SOTA (RSL-SQL, LinkAlign) 未引入 Ontology 层

### G2: Schema Linking 与知识召回割裂（保留）

**证据**：
- RSL-SQL, SchemaGraphSQL 专注 Schema Linking，不召回知识
- Sequeda, ATHENA 用 Ontology，但全量查询不筛选
- 无方法同时做联合召回 + 最小完备集

### G3: Ontology 构建成本阻碍落地（新增）

**证据**：
- ATHENA: 75 概念/289 属性，领域专家数周
- Sequeda: 手工 OWL + R2RML
- OPTIQUE: 半自动但仍需大量人工
